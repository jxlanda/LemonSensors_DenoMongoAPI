API Endpoint

https://deno-mongodb.herokuapp.com/

La API Deno-mongodb proporciona acceso programático para conectarse con la base de datos de MongoDB Atlas creada específicamente para el proyecto LemonSensors.

Crear usuario

                
Mensaje:

{
    "name": "Jose Luis Cuevas",
    "username": "jxlambda",
    "email": "jose.cuevas@tectijuana.edu.mx",
    "password": "1234",
    "tokenfcm": "",
    "role": {
        "id": 0,
        "name": "Administrador"
    }
}

Para crear un usuario se debe realizar una petición POST a la siguiente ruta: user/create
Si el usuario se creo correctamente devolvera el id ($oid) del usuario como respuesta. 


                
Respuesta:

{
    "$oid": "5f9f9ff700d8ebd500bb9724"
}

Cuerpo de la petición

Campo Tipo Descripción
name String Nombre completo
username String Apodo o nombre de usuario
email String Correo electrónico
password String Contraseña en texto plano del usuario
tokenfcm String Token utilizado para enviar Web Push Notifications
role Role Rol o tipo de usuario
id Number Identificador del tipo de usuario o rol
name String Nombre del tipo de usuario o rol

Consultar usuario

Para consultar a un usuario especifico se debe realizar una petición GET a la siguiente ruta: user/:id
Es necesario enviar el id del usuario a buscar como parametros de ruta. Si el id coincide con algún usuario existente regresara los datos de dicho usuario en la respuesta. 

                
                
    Respuesta:
    {
    "_id": {
        "$oid": "5f9fc7e7007e4d710026cb3f"
    },
    "name": "Luis Eduardo Fierro Lopez",
    "username": "FILL16Z",
    "email": "luis.fierro16@tectijuana.edu.mx",
    "password": "$2a$10$X8fC4U8a/cIYlFk1ZxS90uzI5x1RiW6iWXmkrXFJ1eKIr9tA5WlU.",
    "tokenfcm": "",
    "role": {
        "id": 1,
        "name": "Administrador"
    }
    }

maxvlaue Number Valor máximo

Para crear o guardar el sensado se debe realizar una petición POST a la siguiente ruta:
history/create
Si el sensado o historial se creo correctamente devolvera el id ($oid) del historial como respuesta. 

        
        
Respuesta:
{
"$oid": "5fb08cd6kdi344jdf9993d7"
}

Cuerpo de la petición

Campo Tipo Descripción
sensor Sensor Datos mínimos del sensor
id String Id ($oid) del sensor
name String Nombre del sensor
model String Modelo del sensor
measurement String Símbolo de medición
storageplace String Lugar o ubicación del sensor
sensinginseconds Number Número de segundos en los que se esta sensando
maxvalue Number Valor máximo de sensado
minvalue Number Valor mínimo de sensado
user User Usuario del sensor
id String Id ($oid) del usuario
username String Nombre de usuario
email String Correo del usuario
imgurl String Url del avatar del usuario
value Number Valor sensado
datetime Date Fecha y hora del sensado
isEvent Boolean Es una alerta o una lectura normal

{
    "from": "2020-01-01",
    "to": "2020-01-10",

}

Para consultar el historial en un rango de fechas se debe realizar una petición POST a la siguiente ruta: history/range
Si se encuentran coincidencias se enviará como respuesta los registros encontrados. 

        
        
Respuesta:
[
    {
        "_id": {
            "$oid": "5ffad2d20027d1fd009e6aef"
        },
        "sensor": {
            "id": "sfsdf",
            "name": "fsf",
            "modelname": "sfsdf"
        },
        "value": 15,
        "datetime": "2021-01-10T00:00:00.000Z",
        "isEvent": true,
        "date": "2021-01-10"
    }
]

Cuerpo de la petición

Campo Valor Descripción
from Date Es la fecha de inicio
to Date Es la fecha de fin
sensorid String Es el id ($oid) del sensor
